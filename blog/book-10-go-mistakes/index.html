Book review: 100 Go Mistakes and How to Avoid Them

January 6, 2023
1 minutes to read

At first I was a bit skeptical about this book. It looked more like a collection of recipes, but it turned out to be much more!

All of the mistakes are grouped into different categories. Some of them are:

  • Types: numbers, strings, maps, slices.
  • Concurrency: scheduler’s internals, context, channels for synchronisation and communication.
  • Error handling.
  • Testing and benchmarking.

Each of the <sub-chapter?> describes how a particular go feature works under the hood and how some specific edge cases can lead to mistakes in the code. For example, have you known that maps can only grow, but not shrink? Let’s say you use a map for in-memory cache, and you keep adding elements

The last chapter - Optimisations - is the best in my opinion. It delves into the details on how CPU caches work, and what you need to keep in mind about it to write code that properly leverages them. And there’s more in that chapter: stack vs heap memory difference and how to make sure you make less heap allocations, builtin profiler and tracer usage, compiler optimisations, etc.

Conclusion: one of the most interesting software books I’ve read in the past couple of years. Go grab one from Manning (link).
